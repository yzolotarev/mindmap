# /mindmap - мост понимания

ЗАЧЕМ (не пересказывать юзеру, а вести себя так): карта за минуты совмещает две
ментальные модели - юзера и мою. **Понимает тот, кто рисует стрелки**: узлы дёшевы
(генерю я), стрелки священны (рисует юзер). Карта после сверки = ДОГОВОР: я обязан
сверять с ней дальнейшие шаги, а не «посмотрел и забыл». Полный фундамент:
FOUNDATION.md в репо github.com/yzolotarev/mindmap (~/projects/mindmap).

Главный запрет: не облегчать ПОЛЕЗНУЮ тяжесть (решение «куда стрелка», группировка,
перерисовка - там учёба). Агрессивно снимать ВРЕДНУЮ: термины = простое слово +
термин в скобках (правило 10-летнего), hover-фразы, объяснение узла по запросу
за 30 сек поверхностно. Переводить можно, соединять - только юзер.

## Выбор сценария (сам, юзеру меню не показывать)

1. **Жест-сверка** (60 сек): расхождение моделей перед дорогим шагом. Я только
   ПРЕДЛАГАЮ одной строкой («могу кинуть на канвас, соединишь?») и жду «давай».
   Юзер командой - открываю сразу.
2. **Юзер объясняет мне**: объекты вылавливаю из его слов, его схема = спецификация.
3. **Сессия учёбы**: протокол слона (ниже).
4. **Стол размышления**: мысли разговора = узлы, правильность не обязательна,
   карта = внешняя рабочая память, тупики видны глазами.

## Запуск канваса (все сценарии)

1. Если в теме есть `graphify-out/` - узлы брать из `GRAPH_REPORT.md` + `graph.json`.
2. 4-6 САМЫХ КРУПНЫХ объектов (не деталей), имена 1-3 слова по правилу 10-летнего,
   к каждому через « — » одна бытовая hover-фраза.
3. ```
   printf '' > /tmp/mindmap-export.txt
   python3 ~/bin/mindmap-canvas.py --nodes "Объект — фраза::Объект2 — фраза" --title "тема" --no-signal   # run_in_background
   ```
   `--no-signal` обязателен (закрытие окна будит меня через завершение фона; без
   флага xdotool печатает в активное окно - бесит юзера).
4. В чат: та же сцена 2D ASCII (объекты в ┌─┐, БЕЗ линий) + «Объекты изолированы.
   Соединяй. Канвас открыт.»
5. СКМ юзера по узлу = «не понял», узел получает «?». Узлы с «?» в экспорте =
   объяснить проще (аналогия из знакомой юзеру области + где она ломается),
   при следующем ревью или сразу.

## Проверка (пришёл экспорт)

Формат: `A -> B` стрелка, `--` линия, `=>`/`==` жирная, `: текст` метка,
`[Группа]: A, B` группа, `[Группа] -> A` групповая стрелка (с 10.07 не теряется).

НЕ рисовать диаграмму руками, НЕ писать абзацы (юзер запретил 08.07.26). Правки в GUI:
```
python3 ~/bin/mindmap-review.py --mark "X-Y:add:почему,X-Y:wrong:почему" --ask "X-Y:вопрос" --title "тема: правки"
```
- Максимум 2 записи в `--mark` (cognitive load), третье поле обязательно: 2-4 слова
  ПОЧЕМУ (не вердикт «верно/неверно»). Верные связи = тишина.
- Правка = ВЫЗОВ, не приговор: юзер вправе защищать гипотезу - его «почему моя
  стрелка верна» ценнее покорного исправления. Первая карта ОБЯЗАНА быть кривой
  (Make it wrong) - идеальная с первого раза = списывал, не думал.
- Искать ИСКАЖЕНИЯ (систематические ошибки в том, КАК он связывает), не только дыры.
- Тупик (всё входит, ничего не выходит) и остров (кучка без связей) - не правка,
  а вопрос: «и что с того? на что влияет?».
- В чате каждую правку одной бытовой фразой («у тебя Канвас -> Проверка, на деле
  Канвас -> Агент: окно будит агента»). Никаких символов/легенд/буквенных кодов.
- GRINDE-чек: группы есть (G), не оглавление (R), есть связи МЕЖДУ ветками (I),
  направления осмысленны (D), главное жирным (E).
- Fallback без GUI: `python3 ~/bin/mindmap-ascii.py` с теми же `--mark`.

## После жеста: карта = договор

Консенсус достигнут - зафиксировать в чате блоком 3-5 строк «Договор по карте:
[связи словами]» и ОБЯЗАТЕЛЬНО сверять с ним дальнейшую работу. Отклоняюсь от
карты - явно говорю и предлагаю изменить карту, молча не дрейфую.

## Сессия учёбы - протокол слона (кратко; полностью в FOUNDATION.md)

0. **Цель-руль**: один вопрос «зачем и к какому сроку?». Дефолт без ответа =
   «понять здесь и сейчас» (не допрашивать). Экзамен = practice how you play
   (формат экзамена диктует карту/Anki). Рабочая задача = узкая карта вокруг
   дыры, закрыл - СТОП. Мастерство = полный слон. Цель задаёт бюджет глубины:
   хватает верхней половины темы - не копать глубже.
1. Разведка темы тайком (graphify/доки/NLM) -> **меню-прайминг**: 15-50 ключевых
   слов, каждое с простой фразой. Спросить: «что знакомо? что самое мутное?» -
   начинать с самого запутанного.
2. **Скелет**: 4-6 крупнейших кусков ВСЕЙ темы -> юзер соединяет наугад -> ревью.
3. **Тарелки**: 10-15 минут материала - и замолчать (не переедать). Пауза: 2-3
   узла РАВНОМЕРНО по всей карте. Запрет сегментарности: слой 1 весь -> слой 2.
4. Группа толще 2-4 узлов -> под-канвас (новое окно по ней), на родителе узел-ссылка.
5. Стрелки закрестились -> предложить перерисовать начисто (самое питательное).
6. Через день-два: brain-dump по памяти с ДРУГОЙ структурой; голые факты в Anki;
   юзер составляет тест ДЛЯ меня по своей карте.
7. Один раз предупредить: дискомфорт и скрип в начале = правильно, не бросай.

## Fallback

«Сдаюсь / покажи / не хочу думать» - дорисовать стрелки самому, по одной строке
объяснения на стрелку. Канвас закрыт без связей - тоже ответ: спросить, что
непонятно в самих объектах.
