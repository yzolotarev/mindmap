# /mindmap (connect-the-dots)

Метод Джастина Санга: знание = схема связей. Юзер учится, когда САМ рисует стрелки
(Bloom 3-5), а не читает готовое. Работает в обе стороны: я объясняю юзеру ИЛИ юзер
объясняет мне своё ТЗ (его схема = спецификация).

## Фаза 1 - изоляция (Layer 1)

1. Есть `graphify-out/` - сначала прочитать `GRAPH_REPORT.md` (+`graph.json`), узлы оттуда.
2. Выделить 4-7 САМЫХ КРУПНЫХ объектов (не деталей). Имена 1-3 слова, понятны 15-летнему.
3. Очистить `/tmp/mindmap-export.txt` и запустить канвас в фоне:
   `python3 ~/bin/mindmap-canvas.py --nodes "Объект1 — бытовая фраза::Объект2::Объект3" --title "тема"`
   Разделитель узлов `::` (можно `|`). К имени через « — » добавь одну бытовую фразу-объяснение
   (всплывает подсказкой по наведению). Экспорт юзера появится в `/tmp/mindmap-export.txt`.
   Средняя кнопка мыши по объекту у юзера = «не понял», ставит `?` без набора текста -
   такие объекты переформулировать проще на следующей итерации.
4. В чат: 2D ASCII-сцена (объекты в ┌─┐, разбросаны, БЕЗ линий) + «Объекты изолированы. Соединяй. Канвас открыт.»

## Фаза 2 - проверка

Экспорт: `A -> B` стрелка, `A -- B` линия, `=>`/`==` жирная (главная), `: текст` метка, `[Группа]: A, B`.
1. Верное подтвердить коротко. Ошибки/пропуски - максимум 2 за раз через
   `python3 ~/bin/mindmap-review.py --mark "X-Y:add:причина,X-Y:wrong:причина" --ask "X-Y:вопрос почему" --title "тема: правки"`.
2. `--ask` кладёт вопрос «почему?» про ключевую стрелку юзера ПРЯМО НА КАНВАС (оранжевый
   пунктирный узел + линии к обеим сторонам стрелки), продублировать одной строкой в чате.
3. GRINDE-чек: группы (G), не иерархия-оглавление (R), горизонтальные связи (I), направление-логика (D), главное жирным (E).

## Фаза 3 - углубление или фиксация

Предложить одно: раскрыть одну группу в Layer 2 (новый канвас) ИЛИ сохранить схему в Obsidian.

## Fallback

«Сдаюсь / покажи / не хочу думать» - дорисовать стрелки самому, одна строка объяснения на стрелку.

## Правила

- Не рисовать готовые стрелки, пока юзер не попробовал сам (кроме fallback).
- Максимум 7 объектов; 4+ ветвей из одной точки = сгруппировать (chunking).
- Максимум 2 исправления за итерацию (cognitive load).
