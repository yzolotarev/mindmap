# Mindmap - connect-the-dots схема для работы с ИИ

Повышает продуктивность и осознанность при работе с ИИ, а также помогает разобраться
в любой концепции. Вместо того чтобы читать готовое объяснение агента (и забыть его
через час), ты получаешь изолированные объекты и САМ рисуешь связи между ними.
Агент проверяет, исправляет и углубляет.

Основано на методологии обучения Justin Sung: знание = схема связей, а не список
фактов. Настоящее обучение происходит на уровнях «применение / анализ / оценка»
(Bloom 3-5), когда связи строишь сам, а не получаешь готовыми.

## Как это работает

```
Ты: «объясни, как работает sing-box»

Агент:                                Канвас (GUI):
┌────────────┐      ┌──────────┐      открывается окно с теми же
│ Приложение │      │   ТСПУ   │      объектами; ты соединяешь их
└────────────┘      └──────────┘      стрелками, группируешь,
      ┌──────────┐                    подписываешь связи
      │ sing-box │
      └──────────┘                    закрыл окно = схема улетела
┌────────┐        ┌─────────┐         агенту на проверку
│ Сервер │        │ Роутинг │
└────────┘        └─────────┘

Агент: «A -> C верно. C -> B наоборот: ТСПУ смотрит на трафик,
        а не наоборот. Почему ты соединил Роутинг с Сервером?»
```

Работает в обе стороны:

- **Агент объясняет тебе** - изолирует крупные объекты темы, ты соединяешь, он проверяет.
- **Ты объясняешь агенту** - неясное ТЗ раскладывается на объекты, ты рисуешь как оно
  должно работать, твоя схема становится спецификацией.

## Канвас

`mindmap-canvas.py` - однофайловый Tkinter-редактор графов, без зависимостей.

| Действие | Как |
|---|---|
| Двигать объект | ЛКМ + drag |
| Соединить | клик по A, ПКМ по B |
| Стрелка / линия / группа | клавиши C / V / B |
| Метка связи | двойной клик по линии (`!` в конце = жирная, главная) |
| Переименовать | двойной клик по объекту |
| Группа | B + обвести рамкой |
| Undo / Export | Ctrl+Z / Ctrl+E |
| Закрыть окно | авто-экспорт + сигнал агенту |

Запуск:

```bash
python3 mindmap-canvas.py --nodes "Насос::Фильтр::Бак" --title "гидравлика"
```

Экспорт пишется в `/tmp/mindmap-export.txt`:

```
=== NODES ===
  A: Насос
  B: Фильтр
=== CONNECTIONS ===
  A => B : качает      # => жирная стрелка, ": текст" метка
  B -- C               # -- ненаправленная линия
=== GROUPS ===
  [Контур]: A, B
```

## Интеграции

- `integrations/claude-code/` - скилл для Claude Code (`~/.claude/skills/mindmap/`)
- `integrations/pi/` - расширение для [Pi](https://github.com/earendil-works/pi) с
  автотриггером: перехватывает только объяснительные запросы («объясни», «как работает»,
  «хочу чтобы...»), а не каждый ввод
- `integrations/openclaude/` - скилл + hook prompt для OpenClaude

Протокол одинаковый везде:

1. **Изоляция.** Агент выделяет 4-7 самых крупных объектов темы (имена понятны
   15-летнему), открывает канвас и рисует ту же сцену в чате БЕЗ связей.
2. **Проверка.** Ты соединяешь. Агент подтверждает верное, исправляет максимум
   2 ошибки за раз и задаёт один вопрос «почему?» про твою ключевую стрелку.
3. **Углубление.** Одна группа раскрывается в следующий слой детализации, либо
   схема сохраняется заметкой.

Чеклист качества схемы - GRINDE (Justin Sung): Grouped, Reflective, Interconnected,
Non-verbal, Directional, Emphasized.

## Почему это работает

- **Generation effect**: то, что построил сам, помнится кратно дольше прочитанного.
- **Cognitive load**: максимум 7 объектов на сцене, максимум 2 исправления за итерацию.
- **Слои**: сначала общая картина (Layer 1), детали только после неё (Layer 2+).
- **Симметрия**: тот же механизм превращает расплывчатое ТЗ в проверяемую спецификацию,
  до того как агент начал писать код не туда.
